## ☕ Java → Python Notes

As a Java developer, I documented the key differences I noticed along the way.

| Java | Python Equivalent | Notes |
|------|------------------|-------|
| `int x = 5;` | `x = 5` | No type declaration needed |
| `System.out.println()` | `print()` | Much simpler |
| `ArrayList` | `list` | Built-in, no import needed |
| `HashMap` | `dict` | Built-in, no import needed |
| `for(int i=0; i<n; i++)` | `for i in range(n):` | No ceremony |
| `toString()` | `__str__()` | Dunder method |
| Constructor | `__init__(self)` | self = this |
| `try-with-resources` | `with open(...) as f:` | Context manager |
| Checked exceptions | No checked exceptions | Python doesn't force you to declare them |
| Maven/Gradle | pip + requirements.txt | Much simpler |
| JUnit | pytest / unittest | pytest is more concise |
| `// comment` | `# comment` | |
| Varargs `String...` | `*args` | |
| No equivalent | `**kwargs` | Named variable arguments |
| Java Streams | `map()`, `filter()`, generators | Python's version is simpler |
